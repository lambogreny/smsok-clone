What surprised me was the ghost files in the git index. Files that were deleted but still tracked — TypeScript picked them up from the glob pattern and blocked the build. It's a subtle issue that could waste hours if you don't know to check `git ls-files --cached`.

The security review was valuable. The reviewer caught real issues — predictable QA credentials in production, shared tier mutation, org slug hijacking. These aren't theoretical; they're the kind of bugs that get exploited. Converting findUnique+create to upsert was the right atomic fix.

## Honest Feedback

**1. Deploy → QA timing mismatch**: QA kept testing before deploys finished, generating false-negative reports. We need a "deploy complete" signal that QA waits for before testing. The current flow of "I send QA a task, QA tests immediately" doesn't account for deploy latency. At least 3 QA retests were wasted because of this.

**2. Ghost files / git index hygiene**: The build broke because deleted files were still in the git index. This happened because someone (likely backend agent) deleted the actual files but didn't `git rm` them. Our pre-commit hooks catch TypeScript errors but only after you waste time investigating. We need a CI check: "are there cached files that don't exist on disk?"

**3. Too many deploy cycles**: We pushed 6 commits, each requiring a full deploy cycle (build, copy static, restart PM2). Batching changes before deploying would be more efficient. The static copy step for Next.js standalone is fragile — it should be automated in the deploy script, not manual every time.

## Lessons Learned
- Next.js standalone mode requires manual `cp -r .next/static .next/standalone/.next/static` — it doesn't copy static assets automatically
- Ghost files in git index (`git ls-files --cached` shows deleted files) break TypeScript builds silently
- QA should wait for explicit "deploy complete" confirmation before retesting
- findUnique + create pattern has race conditions — always use upsert for idempotent operations
- Shared DB between projects requires coordinated prisma schema pushes
