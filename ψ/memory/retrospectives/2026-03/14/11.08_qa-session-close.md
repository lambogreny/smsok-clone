## Honest Feedback

**Friction 1: Session boundary amnesia.** Every `claude --resume` or new session loses all task context. I reported task #2773 results at least 4 times across different session boundaries. The user kept saying "5 tasks pending" when all 5 were complete. Without persistent task state, we're trapped in a Groundhog Day loop where I re-verify the same completed work. A simple `tasks.json` that persists across sessions would fix this entirely.

**Friction 2: Signal-to-noise ratio.** Roughly 70% of this session's messages were "inbox" with no actionable content. The context window filled with repetitive exchanges instead of productive work. If the orchestration system could batch-check and only wake the agent when there's actual work, session quality would improve dramatically.

**Friction 3: Ambiguity about "fresh run" value.** When the user asked to rerun E2E tests that passed 5 minutes ago with no code changes, it's unclear whether this adds value or wastes compute. A convention like "rerun after deploy" vs "status check" would help me decide whether to actually execute tests or simply report cached results.

## Lessons Learned

1. Persist QA task results to a file (`qa-results.json`) so they survive session resets
2. After completing all assigned work, proactively ask PM for next priorities instead of passively waiting
3. When E2E results are <5 min old with no code changes, report cached results instead of re-running
