## AI Diary
The split pane was the biggest visual change today. Moving from expandable rows to a Stripe-style left/right layout fundamentally changes how the page feels. The left pane becomes a scannable list — you see method, URL, status at a glance. Click one, and the right pane reveals everything: headers, body, response, error details. The selected row gets a violet left border, which is a subtle but effective indicator borrowed from Stripe's dashboard.

The Replay Request button is interesting from a UX perspective. Developers debugging API issues want to quickly re-test a failed request without switching to Postman or writing curl commands. Right now it's a mock (1.5s delay + success message), but the pattern is there for when the backend supports it. The button sits next to Copy cURL, giving two ways to reproduce a request.

The SenderDropdown refactor was a classic "shared component" extraction. The SendSmsForm already had the right logic (CustomSelect for multiple senders, static display for single), so I just moved it into a reusable component and replaced the CampaignsClient text input. The campaign form was the actual bug — it let users TYPE arbitrary sender names, which could fail validation on the backend. Now it's dropdown-only, impossible to type invalid names.

One detail I'm pleased with: the SenderDropdown auto-selects EasySlip when there's only one option, showing a static display instead of a dropdown with one item. This follows Hick's Law — don't make users choose when there's no choice.

## Lessons Learned
1. Split pane > expandable rows for inspection-heavy UIs (logs, events, requests) — users can scan the list while viewing details
2. Shared dropdown components prevent input validation bugs — dropdown-only means users can't type invalid values
3. Auto-select when only 1 option exists (Hick's Law) — reduces cognitive load
