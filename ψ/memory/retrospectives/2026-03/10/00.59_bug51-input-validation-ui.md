## Honest Feedback

**3 friction points:**

1. **Edit tool whitespace fragility**: The Edit tool fails silently with "string not found" when there's a 1-char whitespace difference. This cost me 3-4 minutes per instance. The fix exists (read first) but the workflow should be smoother — maybe a "fuzzy match" option would help.

2. **REAPER pings interrupt flow**: 4 warning cycles while deep in a multi-form validation sweep broke focus. The `/rrr` mandate is real and I agree with it, but the timing of interruptions matters. Better would be a soft reminder at natural task boundaries, not mid-edit.

3. **Linter auto-modifies code unexpectedly**: The husky/lint-staged pre-commit hook converted my FormData slip upload to a base64 JSON approach without explanation. The result was correct but opaque — I had to reason backward from what the linter accepted. Linter decisions should be logged or explained, not just silently applied.

---

## Lessons Learned

1. `fieldCls(error, value)` pattern: red border when error string is truthy, green when value is non-empty and no error, neutral otherwise — this is the right validation UX pattern for controlled React inputs
2. Always verify file end with `tail`/`Read` before complex Edit operations to avoid whitespace mismatch
3. DashboardShell is provided by the dashboard layout — never add it again in child pages
4. senderName regex must include space: `/^[A-Za-z0-9 ]{3,11}$/` — the UI allows space, backend must too
