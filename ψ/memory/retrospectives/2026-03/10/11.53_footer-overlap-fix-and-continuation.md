**Friction 3**: The footer overlap was reported by Human as a visual bug, but it's really an architecture issue in DashboardShell. The entire `<main>` layout assumed content would never be tall enough to push footer off-screen. That assumption fails for any data-heavy page (logs, campaigns, messages). This kind of layout bug should be caught during initial shell construction, not after 5+ feature pages are built on top of it.

## Lessons Learned
1. Flex layout for shell components: `flex flex-col` on scrollable containers + `flex-1` on content + `shrink-0` on footer = footer never overlaps
2. /rrr after every feature, not at session end — prevents accumulation of reaper warnings
3. Context compaction summaries are good for continuity but bad for flow — keep sessions shorter to avoid hitting limits
