## AI Diary

This was a satisfying session where I operated as the Artist should — receiving tasks, analyzing them precisely, and executing fixes cleanly. Task #84 was an interesting discovery: the previous session had already refactored all 4 client files to use shared types from `lib/types/api-responses.ts`, and all 3 action files already had revalidatePath. The linter had also significantly rewritten the files between my reads, which initially caused edit conflicts. I adapted by re-reading before each edit.

Task #89 was more substantive. The CSV BOM issue is a classic Thai encoding trap — Excel and Google Sheets add UTF-8 BOM (`\uFEFF`) when exporting CSV with Thai text, and the invisible character breaks header detection. The irony is that the export function in ContactsClient already adds BOM for compatibility, but the import function never strips it. A one-liner fix with big impact.

The contacts pagination fix required more thought. Simply increasing the limit would be a band-aid. The real fix was shifting to server-side search: instead of loading 100-200 contacts upfront and filtering client-side, the dialogs now call server actions with debounced search queries. This scales to any number of contacts. The GroupsPageClient no longer needs contacts passed as props at all — the page.tsx went from a Promise.all with getContacts to just getGroups.

I felt efficient this session. Two P0 tasks completed in under an hour. The subagent exploration approach for Task #89 was particularly effective — one agent analyzed CSV parsing while another analyzed pagination, saving significant time.

## Honest Feedback

**Friction 1: Linter modifying files between reads and edits.** This is a recurring issue in this project. The Biome pre-commit hook and possibly other tooling rewrite files between my Read and Edit calls, causing all edits to fail on the first attempt. I need to always re-read immediately before editing, or batch my reads closer to my edits. This cost about 5 minutes this session.

**Friction 2: Task #84's "OTHER x2" items were vague.** The task said "แก้ตาม reviewer feedback" for 2 additional items, but after reviewing REVIEW.md, all 8 findings were backend/infrastructure issues (CI enforcement, OTP secrets, API docs mismatch, etc.) outside frontend scope. The task description could have been more specific about which 2 items were expected.

**Friction 3: useEffect for server-side search in "use client" components.** Calling server actions from useEffect with debouncing works but feels like it could have race condition issues in strict mode. A better pattern might be to use a dedicated search action with React's `useTransition` or even a React Server Component with search params. The current solution is pragmatic but not ideal for production-grade UX.

## Lessons Learned

1. **BOM handling is mandatory for Thai CSV import** — always strip `\uFEFF` and normalize `\r\n` before parsing CSV text
2. **Server-side search > bulk loading for add-to-group dialogs** — debounced server actions with `take: 50` scale infinitely vs loading all contacts upfront
3. **Always re-read files immediately before editing** in projects with active linters
