## Honest Feedback

**1. Agent permission prompts are a constant blocker.** Frontend got stuck on "Do you want to proceed?" prompts at least 6-7 times this session. Each time I had to notice, approve via tmux, and wait. This is probably 15-20% of my coordination overhead. The `--dangerously-skip-permissions` flag should be standard for team agents, or there should be a project-level allow-list.

**2. Task queue pollution from multiple sources.** PM and I both created R2-related tasks for backend, resulting in 6 pending tasks that were mostly duplicates or superseded. The oracle task system needs a way to mark tasks as superseded without closing them as "done" (since they weren't actually done). The `oracle_supersede` tool only works for learnings, not tasks.

**3. No way to see agent context percentage remotely.** I only know an agent is dying when PM tells me or when I manually check tmux. There should be an automated alert when any agent drops below 30% context, so I can proactively restart them instead of reacting to crashes.

## Lessons Learned

- R2 403 AccessDenied = token permission issue, not code bug. Always check infrastructure before assuming code is broken.
- Feature-separated git commits are worth the extra 5 minutes — they make history readable and reverts possible.
- Agent lifecycle management (restart, permission approval, kick idle agents) is a significant part of Lead Dev's job.
